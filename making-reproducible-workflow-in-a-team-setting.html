3 Making Reproducible Workflow in a Team Setting

3.1 Introduction

In class, you use RStudio to write code using Quarto documents. To make sure your work is organized, shareable and works on any computer, the best option is to use an R Project stored in Google Drive. You might have different strategies for this but, I want to share my strategy when I work with a team.

3.1.1 R Project

It keeps all your files (datasets, code, results, and images) in one place. Using Google Drive means your project automatically syncs online, so you can open it wherever you are without having errors or editing things.

The important step is organizing everything into folders; for example, with your data: there should be different folders for raw data, cleaned data, and outputs and using relative file paths, codes will run no matter who runs it.

3.2 Steps

I tried to make this Beginner-friendly as possible.

3.2.0.1 Step 1: Install Google Drive Desktop

  1. Download Google Drive for Desktop
  2. Install it and sign in with your DLSU account
  3. Make sure you can see a Google Drive folder on your computer.
    1. You should see My Drive and maybe even Shared Drives

3.2.0.2 Step 2: Create a new R Project in Google Drive

Important: Make sure you have RStudio and R in your own devices.

  1. In RStudio, click File → New Project
  2. Choose New Directory
  3. Choose New Project
  4. Name your project (example: ProblemSet1)
  5. For Location, choose a folder inside Google Drive
    • Example: Google Drive → Shared Drives → Lab Class
  6. Click Create Project

If you are correct, RStudio will reopen and you will see your project name in the top-right corner beside the R inside a blue cube.

3.2.0.3 Step 3: Create folders for organization

In the Files panel (beside the Plots, Packages…):

  1. Click New Folder or the icon of a folder with a green + sign.

  2. Create these folders exactly:

data
scripts
output

  1. Open the data folder and create two more folders inside it:

    • raw for the datasets downloaded from sources

    • cleaned for datasets already cleaned using RStudio (renamed columns, filtering, etc.)

    Your structure should look like this:

ProblemSet1/
  - data/
  --  raw/
  --  cleaned/
  - scripts/
  - output/
  - ProblemSet1.Rproj

Uses:

  • data/raw → NEVER change these
  • data/cleaned → processed/cleaned datasets
  • scripts → R Markdown / Quarto documents (codes)
  • output → graphs, tables, reports

3.2.0.4 Step 4: Add your data

  1. Put your original dataset (CSV, Excel, RData) into:
    • data/raw
  2. Remember: Do not edit raw data files.

3.2.0.5 Step 5: Create an R Markdown document

  1. RStudio: File → New File → R Markdown
  2. Choose HTML
  3. Click OK
  4. Save the files as:
    • cleaning.Rmd
    • descriptives.Rmd, etc.
  5. Make sure you save these .Rmd files in the scripts folder
  6. In your YAML Header:
---
title: "Problem Set 1 - Cleaning Datasets"
author: "All member names"
date: "`r Sys.Date()`"
output:
  html_document:
    toc: true
    theme: united
knitr:
  opts_chunk:
    warning: false
    message: false
---

3.2.0.6 Step 6: Set up packages (VERY IMPORTANT STEP!)

At the top of your R Markdown document, after your YAML Header, add this code chunk:

packages <- c("tidyverse") # all packages needed

for (pkg in packages) {
  if (!require(pkg, character.only = TRUE)) {
    install.packages(pkg)
  }
  library(pkg, character.only = TRUE)
}

This is important because if a teammate opens your project on another computer, R will automatically install any missing packages.

3.2.0.7 Step 7: Load data using relative paths

To load data from the data/raw folder:

df <- read.csv("data/raw/my_data.csv")

Important:

  • Never use paths like C:/Users/...
  • Check! Always start paths from the project folder

This ensures the project will work on any computer.

3.2.0.8 Step 8: Clean data and save it

After cleaning your data, save it to the data/cleaned folder:

clean_df <- df %>%
  filter(!is.na(column))

write.csv(clean_df, "data/cleaned/clean_df.csv", row.names = FALSE)

3.2.0.9 Step 9: Run code

  • Click the green play button on a code chunk to run the code chunk
  • Output will appear below the chunk

If there is an error, the green vertical line at the right side of the chunk will become red. An error message will appear. Read it carefully.

  • Issue might be objects not found → Check Environment
  • Typos and file names → Check code chunk

3.2.0.10 Step 10: Save Plots, Descriptive Statistics, and Regression Results

3.2.0.10.1 a. Save Plots

Save plots to the output folder

# Option 1: using ggplot2
ggplot(clean_df, aes(x = x, y = y)) +
  geom_point()

ggsave("output/scatterplot.png")

# Option 2: using base R
png("output/scatterplot.png")   # Open PNG device
plot(clean_df$x, clean_df$y,
     main = "Scatter Plot of x vs y",
     xlab = "X values",
     ylab = "Y values",
     pch  = 19, col = "blue")
dev.off()  # Close the PNG device
3.2.0.10.2 b. Save Descriptive Statistics
3.2.0.10.2.1 - Base R summary table as CSV
dsc_stat <- summary(clean_df)
sum_df   <- as.data.frame(dsc_stat)
write.csv(sum_df, "output/descriptive_stats_base.csv", row.names = TRUE)
3.2.0.10.2.2 - Using stargazer for a nice formatted table

Include stargazer package at the start of the R Markdown document

library(stargazer)

stargazer(clean_df[, c("x", "y")],
          type  = "text",            # "html" also works
          title = "Descriptive Statistics",
          out   = "output/descriptive_stats_stargazer.txt")

I like the look of stargazer tables because they are easier to read

3.2.0.10.3 c. Save Regression Results
# Run a linear regression
model <- lm(y ~ x, data = clean_df)

# 1) Save coefficients as CSV (Base R)
coef_df <- as.data.frame(coef(model))
write.csv(coef_df, "output/reg_coef.csv", row.names = TRUE)

# 2) Save full regression summary as text (Base R)
sink("output/reg_summary.txt")
summary(model)
sink()

# 3) Save regression table using stargazer
stargazer(model,
          type  = "text",            # "html" also works
          title = "Regression Results",
          out   = "output/reg_star.txt")

Important:

  • Always save outputs in output/
  • stargazer tables are optional but they just look really good.

3.2.0.11 Step 11: Knit your R Markdown document

  1. Click Knit at the top
  2. An HTML file is created
  3. This can be shared with teammates

3.2.0.12 Step 12: Save and close

  1. Save all files
  2. Close RStudio
  3. Google Drive automatically syncs everything

3.3 Rules

  • Never move the project folder outside Google Drive
  • Never rename folders after starting
  • Never edit files in data/raw
  • Always use relative paths
  • All code you do go in R Markdown files

3.4 Closing Message

You do not have to do this, it is just something I want to share regarding how I do my work in a team and hope to share it with you. I think it is useful especially in the future.
